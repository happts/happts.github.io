使用GitHub page搭建博客的经历

发表于 2019-08-22 更新于 2019-08-29

跟着教程搭起来一个简单的博客,顺便熟悉一下新的粉粉嫩嫩的电脑,做点什么东西.

第一篇进行所谓的技术总结(估计搭博客的教程用一次就再也用不到了),旧教程总有过时的部分,希望这篇能给其他人带来点作用.(果然,贴了少女贴纸又加膜的打字手感一点都不好,还是撕掉打字爽)

快速开始

  1. 去 GitHub 注册一个账号(默认大家都有了)

  2. 然后去新建一个仓库

  3. 仓库名字是 你的GitHub账户.github.io ,其他全部默认,创建

  4. 然后去本仓库的设置页settings

  5. 往下拉,找到GitHubPage

    (设置完成了的话应该是上图所示的样子)

    第一次建时去选择一个主题 Choose a theme (选默认的就行,反正后面也要改)

  6. 选择完成后,开始你的第一次commit(随便输入,诸如init,first commit之类的)

  7. 建站结束,访问 你的id.github.com 就能看到了.

  8. 第一篇hello world 等着你.

前期准备

假设你也用的是一台全新的macOS电脑,咱们从头开始.

Terminal 设置

总得要从本地管理你的博客,修改的差不多了再上传到GitHub.先把工具整顺手了咱们再开始.

  1. 去App Store安装 Xcode ,打开,等待安装成功 (如果你不需要Xcode可以跳过)
  2. 反正我装了,跳过Xcode会缺Command Line Tools,里面包含Git等命令行工具,不保证下面的步骤有没有效
  3. 打开 Terminal (终端), 弹出小黑框

安装homebrew(点我进官网)

这是个包管理工具,似乎是能补全⤴️没装Xcode缺失的东西

  1. 在Terminal中输入

等待安装完成

  1. 安装完成后在Terminal中输入 git --version

    看看有没有效果,如果有

    git version 2.20.1 (Apple Git-117)

    类似这样的文字输出到屏幕上那就有Git了

  2. 话说你用macOS还不下载一个Xcode??? ,就不该考虑没有Xcode的情况(溜了溜了)

Terminal切换zsh

zsh + oh my zsh 会比 默认的 bash 好用点,你有自己的习惯这部分可以跳过

  1. 依旧在Terminal中,输入 chsh -s /usr/local/bin/zsh 切换到zsh

  2. 下载oh my zsh 输入

  1. 等待安装完成后关了Terminal再重进来, 左边应该会出现一个小箭头,这就切换成功了

GitHub Desktop + 不使用主题配置网站

下载一个GitHub Desktop ,管理起来可能会方便一点

下载安装,登录,然后把自己的博客仓库clone到本地(自己选路径)

  1. 去finder里面康康这个文件夹,这就是你的网站啦.

  2. _config.yml 是配置文件,具体配置功能自己去找

  3. index.md 或是 index.html 就是你的主页了, happts.github.io 访问你的网址首先出现的就是index.html

  4. 其余的网页或者资源就自己放吧, 记得使用相对路径去访问资源就好啦

    比如我在第一篇文章中添加的一张sans ![sans](images/sans.jpg)

    (无论加什么东西都记得放到一个文件夹里)
