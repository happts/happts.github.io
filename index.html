自定义UISegmentedControl

发表于 2019-08-23

UI要求实现一个性别选择,很明显就是UISegementedControl,只是外观需要自己修改实现一下,第一反应就想着去使用UISegementedControl了(当然你也可以自定义View去实现)

本文很浅薄地接涉及到了KVO ,KVC 等Runtime机制

UI如上图所示,任务清单如下

  1. Segment
    1. 文字
    2. 边框
    3. 背景
    4. (选中和未选中两个状态)
  2. Segment之间的间距

消除原生效果

把蓝色前景和白色背景都改掉

Segment

title

title自定义比较简单,查一下UISegementedControl的文档,调用接口就完事了

边框&背景

既然我们用的是UISegementedControl,盲猜它的subviews也是UISegement,可惜查UIKit里面并没有这个类,看来这是个私有类,不能用.(现在只是猜测subview是什么类,后文去验证)

但是不管它是什么类,它都是UIView的子类就对了,UIView能设置边框border和背景background,那就把它的子类当成UIView用就完事了

现在边框和背景都设置了,但是不能随着状态改变,后续需要绑定状态

间距

查文档,UISegementedControl还是不能直接设置间距,只有一个设置分割图片的方法

那就自己生成图片,设置好尺寸,实现间距自定义

依照api的功能,根据两边的不同状态设置不同的分割图片,这里全部设置为透明的图片

添加观察

做到这里,只是一个静态的选择控件,还需要根据选中的Segment去改变UI

初步设想

观察整体: UISegmentedControl.selectedSementIndex

这个属性是可以直接访问到的,只要这个有变化,咱们就去重新设置subviews的外观

既然有索引index,根据index去找到subviews中选中的Segment,重新设置外观就完成了

(但是索引对应的UISegment和你直观的思想是冲突的,处理起来逻辑会很混乱)

后续的想法:

观察局部: Segment.isSelected

如果说每一个Segment本身都有一个属性isSelected,那直接监听此属性,让Segment自己改变外观就好了. 接下来就需要去验证我们的想法

UISegementedControl的subview是什么

新建一个playground,进行验证

确实是一个UISegment,也确实是一个私有类,文档没有给出此类的任何接口,无法直接调用

Segment 是否有 isSelected 属性

UISegment 是私有类,没有文档可看,只能借助runtime机制去看一下它有什么属性
成员变量与属性的区别

selected就是我们需要的属性,打印出来看看是不是布尔型(其实是Int)

由此可知,选中为1,未选中为0

添加监听

再新建一个playground,做一个小demo,测试一下

封装控件

既然功能上可行,那就封装好一个自定义UISegementedControl,方便使用,其实仅仅是对外观的修改,功能没有任何变化

和UISegmentedControl的调用别无二致,最后的效果如下

仅仅是一个简单的控件,没有考虑到后续insertSegment的情况,本文中也没有用条纹背景,用灰色背景替代,有兴趣的小伙伴可以自己去实现完善.

使用GitHub page搭建博客的经历

发表于 2019-08-22 更新于 2019-08-23

跟着教程搭起来一个简单的博客,顺便熟悉一下新的粉粉嫩嫩的电脑,做点什么东西.

第一篇进行所谓的技术总结(估计搭博客的教程用一次就再也用不到了),旧教程总有过时的部分,希望这篇能给其他人带来点作用.(果然,贴了少女贴纸又加膜的打字手感一点都不好,还是撕掉打字爽)

快速开始

  1. 去 GitHub 注册一个账号(默认大家都有了)

  2. 然后去新建一个仓库

  3. 仓库名字是 你的GitHub账户.github.io ,其他全部默认,创建

  4. 然后去本仓库的设置页settings

  5. 往下拉,找到GitHubPage

    (设置完成了的话应该是上图所示的样子)

    第一次建时去选择一个主题 Choose a theme (选默认的就行,反正后面也要改)

  6. 选择完成后,开始你的第一次commit(随便输入,诸如init,first commit之类的)

  7. 建站结束,访问 你的id.github.com 就能看到了.

  8. 第一篇hello world 等着你.

前期准备

假设你也用的是一台全新的macOS电脑,咱们从头开始.

Terminal 设置

总得要从本地管理你的博客,修改的差不多了再上传到GitHub.先把工具整顺手了咱们再开始.

  1. 去App Store安装 Xcode ,打开,等待安装成功 (如果你不需要Xcode可以跳过)
  2. 反正我装了,跳过Xcode会缺Command Line Tools,里面包含Git等命令行工具,不保证下面的步骤有没有效
  3. 打开 Terminal (终端), 弹出小黑框

安装homebrew(点我进官网)

这是个包管理工具,似乎是能补全⤴️没装Xcode缺失的东西

  1. 在Terminal中输入

等待安装完成

  1. 安装完成后在Terminal中输入 git --version

    看看有没有效果,如果有

    git version 2.20.1 (Apple Git-117)

    类似这样的文字输出到屏幕上那就有Git了

  2. 话说你用macOS还不下载一个Xcode??? ,就不该考虑没有Xcode的情况(溜了溜了)

Terminal切换zsh

zsh + oh my zsh 会比 默认的 bash 好用点,你有自己的习惯这部分可以跳过

  1. 依旧在Terminal中,输入 chsh -s /usr/local/bin/zsh 切换到zsh

  2. 下载oh my zsh 输入

  1. 等待安装完成后关了Terminal再重进来, 左边应该会出现一个小箭头,这就切换成功了

GitHub Desktop + 不使用主题配置网站

下载一个GitHub Desktop ,管理起来可能会方便一点

下载安装,登录,然后把自己的博客仓库clone到本地(自己选路径)

  1. 去finder里面康康这个文件夹,这就是你的网站啦.

  2. _config.yml 是配置文件,具体配置功能自己去找

  3. index.md 或是 index.html 就是你的主页了, happts.github.io 访问你的网址首先出现的就是index.html

  4. 其余的网页或者资源就自己放吧, 记得使用相对路径去访问资源就好啦

    比如我在第一篇文章中添加的一张sans ![sans](images/sans.jpg)
